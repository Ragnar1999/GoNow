Single-page app bootstrapped from `src/main.tsx` into a React 19 root wrapped in `<StrictMode>`. `src/App.tsx` composes the global layout — `BrowserRouter` with three routes (`/`, `/player/:pin`, `/favorites`) — and provides two cross-cutting contexts: a single `QueryClient` instance (staleTime 30s, retry 1) via `@tanstack/react-query` and the router. Feature boundaries are organized by concern:
- `src/api/client.ts` — the only HTTP boundary; a single Axios instance pointing at `http://localhost:8000/api` exposes typed functions (`searchPlayers`, `getPlayer`, `getPlayerTournaments`, `sendChatMessage`) alongside shared TypeScript interfaces (`PlayerSummary`, `PlayerDetail`, `SearchResponse`, `ChatMessage`, …). Pages and components import these functions rather than calling axios directly.
- `src/pages/*` — route-level containers that call the API layer and render presentational UI.
- `src/components/*` — reusable UI fragments (`Navbar`, floating `ChatWidget`).
- `src/hooks/useFavorites.ts` — local state hook persisting favorite player pins to `localStorage` under key `gonow_favorites`; pages consume it instead of managing favorites themselves.

Dependency direction is strictly one-way: pages → hooks/components → api/client → axios. No component imports another page, and no page imports another page. Styling is CSS custom properties in `src/index.css` plus inline `Record<string, React.CSSProperties>` style objects inside components (e.g. ChatWidget's `styles` constant). Build/dev tooling is Vite 8 with the official React plugin; linting uses oxlint via `pnpm lint`.
