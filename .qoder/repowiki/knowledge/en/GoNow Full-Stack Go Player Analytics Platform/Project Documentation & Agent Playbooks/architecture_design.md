This leaf module is a flat collection of four Markdown files that serve as the single source of truth for human readers (developers, reviewers, and LLM agents). `ARCHITECTURE.md` provides the top-level system diagram (React frontend → FastAPI backend → OpenRouter tool-calling agent → EGD GraphQL API), tech stack, project layout, data flow, and key design decisions. `AGENT_DESIGN.md` documents the chat agent's chosen approach (OpenRouter native tool calling over LangGraph/ReAct libraries), sandbox research, model configuration, and the five tool schemas (`search_player`, `get_player_details`, `get_player_rating_history`, `get_player_games`, `compare_players`). `AGENTS.md` is the operational playbook — quick-start Makefile targets, env vars, code conventions, endpoint table, and file index used by coding agents. `EGD_API.md` is an introspection-derived reference of the EGD GraphQL v2026.02 schema (queries, types, enums) consumed by `backend/app/services/egd_client.py`. There are no code dependencies between these files; they are cross-referenced by path to guide development in `backend/` and `frontend/`.
